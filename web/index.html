po0 防火墙自动加白 · Surge 模块

自动把设备当前出口 IP 加入 po0 防火墙白名单。支持多台机器（多 token）、蜂窝网络优化、坑位记账，加白后才能连上开启了防火墙的 po0 机器。

⚡ 一键安装到 Surge

需要 Surge iOS / Surge Mac 已安装。点击后 Surge 会弹出模块安装确认。

https://raw.githubusercontent.com/reallinzc/po0fw/main/po0-firewall-whitelist.sgmodule

按钮无效时，复制上面的地址到 Surge → 模块 → 从 URL 安装。

1获取 token

在 po0 控制台的机器详情页开启「防火墙加白」功能，复制生成的 API token（pgnfw_ 开头）。一台机器对应一个 token。

2安装模块并填入 token

点上方按钮安装模块，安装时（或之后在模块设置里）填写参数 tokens：

单台机器： pgnfw_xxxxxxxx
多台机器： pgnfw_xxxxxxxx,pgnfw_yyyyyyyy

多个 token 用英文逗号分割，脚本会依次为每台机器加白。

3完成，看面板

之后无需任何操作：切换网络时立即、以及每 2 分钟自动检查一次，出口 IP 变了才会真正写入白名单。Surge 首页会多一个面板：

po0 加白 1/1 · 出口 203.0.113.10
#1 ✅ 3/5
    203.0.113.10 ←
    198.51.100.7 📶
    192.0.2.15
标记含义
←当前出口 IP

🧠 设计说明

  • 省坑位：每次先只读查询，当前出口已在白名单就不写入。白名单上限 5 个且 API 无删除能力，省着用很重要。
  • 蜂窝优化：蜂窝出口是 CGNAT，IP 变化频繁。自动加白在 24 小时内最多为蜂窝消耗 2 个坑位，超限后暂停并通知；点面板手动刷新不受限，需要连机器时点一下即可加白。
  • 安静运行：状态用 Surge 持久化存储记账，只在失败、限频或消耗新坑位时才推送通知。
  • 注意：坑位满后旧 IP 需到 po0 控制台手动删除；模块不显示、也不上传你的 token。

❓ 常见问题

Q: 走代理时会把代理服务器 IP 加白吗？
不会。模块自带 DOMAIN,console.po0.com,DIRECT 规则，API 请求始终直连，看到的是设备真实出口 IP。

Q: 家宽换 IP 后多久生效？
网络切换事件是即时的；纯粹的公网 IP 变更最迟 2 分钟内被 cron 捕捉。

Q: 支持几台 po0 机器？
不限，tokens 逗号分割即可，面板按 #1 #2 … 分行显示。
